# backend/orchestrator

本目录用于耦合主系统，是后端主流程调度入口。

## 1. 职责
- 统一接收前端配置并管理系统状态。
- 调度图像识别模块、PID 反馈模块、泵硬件交互模块。
- 管理初始化流程、控制循环流程、停止流程和异常流程。
- 向前端输出统一系统快照（识别状态、泵状态、控制状态、错误信息）。

## 2. 实时视频模式流程
1. 接收前端配置。
2. 准备实时视频输入。
3. 启动图像识别模块。
4. 连接泵硬件并完成探测/回读。
5. 下发并校验初始 Q1/Q2。
6. 初始化完成后，用户点击开始运行。
7. 周期读取识别结果并调用 PID。
8. 根据 PID 输出执行：
- 冻结：保持当前流速。
- 建议停机：执行停泵保护并进入异常/停止状态。
- 正常：下发新流速并回读校验。
9. 前端周期刷新识别、泵和控制状态。
10. 用户停止时，停止控制循环、停泵、停止识别，进入 `STOPPED`。

## 3. 本地视频模式流程
1. 接收前端配置。
2. 准备本地视频输入。
3. 启动识别处理并输出识别结果。
4. 只做识别与显示，不做泵控制下发。
5. 视频处理结束后进入 `STOPPED`。

## 4. 控制循环流程（实时模式）
1. 定时获取识别快照。
2. 构造 PID 输入（识别指标 + 目标参数 + 泵状态 + dt）。
3. 调用 PID 反馈模块。
4. 判断输出分支：
- `freeze_feedback=True`
- `suggested_stop=True`
- 正常控制输出
5. 执行泵下发与回读校验（正常分支）。
6. 更新系统快照供前端刷新。

## 5. 异常处理流程
- 识别无效：不更新控制输出，保持当前泵流速。
- PID 冻结：保持当前泵流速并显示冻结原因。
- PID 建议停机：调用泵模块停泵并进入 `ERROR` 或 `STOPPED`。
- 泵通信异常：执行停泵保护并进入 `ERROR`。
- 用户主动停止：安全停机后进入 `STOPPED`。

## 6. 状态流转
- `IDLE`：空闲未配置。
- `CONFIGURED`：已接收参数配置。
- `VIDEO_READY`：视频源已准备。
- `INITIALIZING`：初始化中（设备连接/回读/初始下发）。
- `INITIALIZED`：初始化完成，可开始运行。
- `RUNNING`：运行中。
- `PAUSED`：暂停中。
- `STOPPING`：停止中。
- `STOPPED`：已停止。
- `ERROR`：异常状态。

推荐主路径：
`IDLE -> CONFIGURED -> VIDEO_READY -> INITIALIZING -> INITIALIZED -> RUNNING -> STOPPING -> STOPPED`

## 7. 目录关键文件
- `service.py`：耦合层主服务实现（流程调度与循环执行）。
- `flow.py`：系统流程骨架定义（状态 + 阶段方法）。
- `models.py`：系统配置与快照数据模型。
- `state.py`：系统状态枚举。
- `config.py`：耦合层运行参数配置。
- `vision_adapter.py`：视觉适配器协议与通用封装。
