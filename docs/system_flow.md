# 系统运行流程说明

本文档基于《微流控液滴直径反馈控制系统框架说明》，仅整理系统流程与调用关系，不展开图像识别算法、PID 算法细节或泵通信协议细节。

## 1. 系统总体流程
1. 前端进入参数设定页面。
2. 用户输入目标液滴平均直径、像素转微米系数和基础运行参数。
3. 前端进入视频来源选择页面。
4. 用户选择实时摄像头或本地视频文件。
5. 前端进入初始化参数页面。
6. 用户输入初始 Q1、Q2 流速。
7. 前端将完整配置提交给后端耦合部分（orchestrator）。
8. 耦合部分根据视频来源进入不同流程。

## 2. 实时视频模式流程
1. 耦合部分接收前端配置。
2. 耦合部分准备实时视频输入。
3. 耦合部分调用图像识别模块开始识别。
4. 耦合部分调用泵硬件交互模块连接泵硬件。
5. 泵硬件交互模块完成设备探测、状态回读和通信确认。
6. 耦合部分下发初始 Q1、Q2 流速。
7. 泵硬件交互模块回读校验初始化参数。
8. 初始化完成。
9. 用户点击开始运行。
10. 耦合部分进入控制运行状态。
11. 耦合部分周期性获取图像识别结果。
12. 图像识别部分输出液滴总数、平均直径、单胞率和有效性状态。
13. 耦合部分将平均直径、目标直径、当前泵状态传给 PID 反馈模块。
14. PID 反馈模块计算 Q1、Q2 控制建议。
15. 耦合部分判断 PID 输出是否冻结或建议停机。
16. 若反馈冻结，保持当前泵流速。
17. 若建议停机，耦合部分调用泵硬件交互模块停泵并进入异常或停止状态。
18. 若反馈正常，耦合部分调用泵硬件交互模块下发新的 Q1、Q2 流速。
19. 泵硬件交互模块回读校验下发结果。
20. 耦合部分更新系统状态。
21. 前端监控页面周期性刷新识别结果、泵状态和控制结果。
22. 用户点击停止后，耦合部分停止控制循环。
23. 耦合部分调用泵硬件交互模块停止泵。
24. 耦合部分停止图像识别流程。
25. 系统进入停止状态。

## 3. 本地视频模式流程
1. 耦合部分接收前端配置。
2. 耦合部分准备本地视频输入。
3. 耦合部分调用图像识别模块开始处理视频。
4. 图像识别部分输出液滴总数、平均直径、单胞率和有效性状态。
5. 耦合部分更新识别状态。
6. 前端监控页面显示视频画面和识别结果。
7. 本地视频处理结束后，系统进入停止状态。

## 4. 状态流转
- `IDLE`：系统空闲，尚未配置。
- `CONFIGURED`：前端已提交基础参数。
- `VIDEO_READY`：视频来源已准备完成。
- `INITIALIZING`：正在执行设备连接、状态回读和初始流速下发。
- `INITIALIZED`：初始化完成，可以开始运行。
- `RUNNING`：系统正在运行。
- `PAUSED`：系统暂停。
- `STOPPING`：系统正在停止。
- `STOPPED`：系统已停止。
- `ERROR`：系统出现异常。

推荐主流转路径：
`IDLE -> CONFIGURED -> VIDEO_READY -> INITIALIZING -> INITIALIZED -> RUNNING -> STOPPING -> STOPPED`

暂停恢复路径：
`RUNNING -> PAUSED -> RUNNING`

异常路径：
`RUNNING/INITIALIZING -> ERROR`（必要时执行停泵保护）

## 5. 控制循环流程（实时模式）
1. 按 `control_interval_ms` 周期触发。
2. 获取识别快照（平均直径、液滴数、单胞率、有效性）。
3. 判断是否满足控制条件。
4. 组装 PID 输入（识别指标 + 目标参数 + 当前泵状态 + dt）。
5. 调用 PID 模块计算控制建议。
6. 处理三种结果分支：
- `freeze_feedback=True`：保持当前流速，仅更新显示状态。
- `suggested_stop=True`：执行停泵保护并转入 `ERROR` 或 `STOPPED`。
- 正常控制：下发新的 Q1/Q2 并回读校验。
7. 更新系统快照供前端刷新。

## 6. 异常处理流程
### 6.1 识别结果无效
- 耦合部分不下发新的控制结果。
- 保持当前泵流速。
- 前端显示识别无效或样本不足。

### 6.2 PID 冻结反馈
- 耦合部分保持当前泵流速。
- 前端显示冻结原因。

### 6.3 PID 建议停机
- 耦合部分调用泵硬件交互模块停泵。
- 系统进入 `ERROR` 或 `STOPPED`。
- 前端显示停机原因。

### 6.4 泵通信异常
- 耦合部分调用泵硬件交互模块执行停机保护。
- 系统进入 `ERROR`。
- 前端显示泵通信异常。

### 6.5 用户主动停止
- 耦合部分停止控制循环。
- 停止泵。
- 停止图像识别。
- 系统进入 `STOPPED`。

## 7. 模块调用关系
- 前端：负责参数输入、操作触发、状态展示。
- 耦合部分（orchestrator）：负责全流程调度与状态管理。
- 图像识别（vision）：负责识别处理并输出识别指标。
- PID 反馈（pid_control）：负责根据误差计算控制建议。
- 泵硬件交互（pump_hardware）：负责连接、下发、回读、校验与停泵保护。

调用方向：
`frontend -> orchestrator -> vision / pid_control / pump_hardware`

前端不直接调用图像识别、PID 或泵硬件通信。
